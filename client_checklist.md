## 2. קבצים ומיקומים לעריכה

### 2.1 משתני סביבה (`.env` / פלטפורמת Deploy)

אלו **לא בקוד** — מגדירים בזמן deploy (קובץ `.env`, Render dashboard, וכו').

| משתנה | תיאור | דוגמה |
|--------|--------|-------|
| `FB_EMAIL` | אימייל פייסבוק | `client@gmail.com` |
| `FB_PASSWORD` | סיסמת פייסבוק | `••••••` |
| `OPENAI_API_KEY` | מפתח OpenAI API | `sk-proj-...` |
| `TELEGRAM_BOT_TOKEN` | טוקן בוט טלגרם | `123456:ABC-DEF...` |
| `TELEGRAM_CHAT_ID` | Chat ID (אפשר כמה מופרדים בפסיק) | `-100123456789` |
| `TIMEZONE` | אזור זמן | `Asia/Jerusalem` |
| `QUIET_HOURS` | שעות שקטות (אופציונלי) | `02:00-07:00` |
| `INTERVAL_MINUTES` | תדירות סריקה בדקות | `10` |
| `BLOCK_KEYWORDS` | מילים חוסמות (אופציונלי, מופרד בפסיק) | `ספאם,פרסומת` |
| `PRE_FILTER_KEYWORDS` | מילות מפתח לסינון מוקדם (מופרד בפסיקים). ריק = אין סינון מוקדם | `בוט,אוטומציה,מתכנת` |
| `CLASSIFICATION_CRITERIA` | פרומפט סיווג AI (טקסט מלא). ריק = חובה להגדיר דרך פאנל | ראו דוגמה למטה |
| `PANEL_TOKEN` | טוקן אימות לפאנל ההגדרות (להמציא מחרוזת חזקה) | `mySecretToken123!` |
| `ERROR_CHAT_ID` | Chat ID לשליחת שגיאות טכניות (אופציונלי — בד"כ ה-ID של המפתח) | `-100987654321` |
| `DEVELOPER_CHAT_ID` | Chat ID של המפתח לפקודות `/developer_usage` ו-`/debug` (אופציונלי) | `123456789` |
| `MAX_GROUPS` | מגבלת קבוצות ללקוח (0 = ללא הגבלה) | `10` |
| `LOG_LEVEL` | רמת לוגים | `INFO` |

**קובץ התייחסות:** `render.yaml` — תבנית deploy ל-Render (שורות 5-27). לעדכן ברירות מחדל אם צריך.
> **הערה:** `PANEL_PORT` לא צריך ב-Render — השירות מאזין אוטומטית על `PORT` שמוגדר ע"י Render. מקומית ברירת מחדל 8080.

> **חשוב:** `PANEL_TOKEN` מגן על פאנל ההגדרות. בלי ערך — הפאנל פתוח לכולם! להגדיר ב-Render dashboard (לא בקוד).

> **טיפ:** `ERROR_CHAT_ID` — הגדירו את ה-Chat ID **שלכם** (המפתח) כדי שהתראות על שגיאות טכניות יגיעו אליכם ולא ללקוח. הלקוח מקבל רק לידים ל-`TELEGRAM_CHAT_ID`, שגיאות הולכות ל-`ERROR_CHAT_ID`.

### 2.2 קריטריוני סיווג AI — **הכי חשוב!**

**דרכים להגדרה (לפי סדר עדיפות):**
1. **פאנל הגדרות** (פורט 8080) — סעיף "קריטריוני סיווג" (נשמר ב-DB)
2. **משתנה סביבה** `CLASSIFICATION_CRITERIA` — טקסט הפרומפט המלא
3. אם שניהם ריקים — הבוט **לא יסווג** (חובה להגדיר!)

זה הלב של הבוט — מה נחשב "ליד רלוונטי". **חובה** להתאים ללקוח.

**דוגמת פרומפט עבור פרילנסר מפתח (הפרומפט המקורי של הפרויקט):**
```
אתה מסנן לידים לפרילנסר מפתח ישראלי.
תפקידך: לזהות פוסטים בקבוצות פייסבוק שבהם עסק קטן או בעל עסק קטן מחפש מפתח שיבנה לו משהו.

כלל מפתח — זהה את תפקיד הכותב:
- אם הכותב הוא **מפתח/יזם** שמציג את העבודה שלו ("אני בונה...", "פיתחנו כלי...", "אני מפתח...") — תמיד לא רלוונטי, גם אם מוזכרים עסקים קטנים.
- רק אם הכותב הוא **לקוח** שמחפש מישהו שיבנה/יפתח עבורו — רלוונטי.

רלוונטי = כן אם:
- עסק קטן מחפש מפתח / פרילנסר / מתכנת לפרויקט
- בעל עסק קטן מחפש מישהו לבנות לו אתר, בוט, אוטומציה, כלי AI, ווב אפ
- הצעת עבודה מעסק קטן לפרילנסר

רלוונטי = לא אם:
- הכותב מציג עצמו כמפתח / יזם / בונה ("אני מפתח...", "בנינו כלי...", "עובד על...")
- מפתח שמפרסם מוצר או שירות שלו
- מישהו מחפש עבודה (הם המועמד, לא הלקוח)
- חברה גדולה או סטארטאפ מגויס מחפשים עובד שכיר
- שאלה טכנית כללית ללא כוונת שכירה
- שיתוף מאמר / כלי / תוכן
- דיון כללי על AI או פיתוח
```

**דוגמה ללקוח שהוא מעצב גרפי:**
```
אתה מסנן לידים למעצב גרפי פרילנסר.
תפקידך: לזהות פוסטים בקבוצות פייסבוק שבהם מישהו מחפש מעצב.

רלוונטי = כן אם:
- מישהו מחפש מעצב / מעצבת גרפית
- צריך עיצוב לוגו, כרטיס ביקור, פלאייר, באנר
- מחפש מישהו לעצב לו אתר / פוסטים לרשתות חברתיות
...
```

### 2.3 קבוצות פייסבוק

**אין ברירות מחדל** — חובה להגדיר קבוצות לפני הפעלה.

**דרכים להגדרה:**
1. **פאנל הגדרות** (פורט 8080) — הדרך הנוחה ביותר
2. **טלגרם** — פקודת `/add_group <url>`
3. **משתנה סביבה** `FB_GROUPS` — מופרד בפסיק

> **חשוב:** URLים חייבים להיות של `m.facebook.com` (לא `www`). המערכת ממירה אוטומטית `www` ל-`m`.

> **שים לב:** בלי קבוצות מוגדרות, הבוט לא יסרוק כלום. הפאנל יציג הודעה מתאימה.

### 2.4 מילות מפתח לסינון מוקדם

**דרכים להגדרה (לפי סדר עדיפות):**
1. **פאנל הגדרות** או פקודות טלגרם `/add_keyword`, `/remove_keyword` (נשמר ב-DB)
2. **משתנה סביבה** `PRE_FILTER_KEYWORDS` — מופרד בפסיקים
3. אם שניהם ריקים — אין סינון מוקדם (כל הפוסטים עוברים ל-AI)

**דוגמה למשתנה סביבה עבור מעצב גרפי:**
```
PRE_FILTER_KEYWORDS=מעצב,עיצוב,לוגו,גרפי,באנר,פלאייר,כרטיס ביקור,ברושור,מיתוג,דרוש מעצב
```

**דוגמה עבור פרילנסר מפתח (המילים המקוריות של הפרויקט):**
```
PRE_FILTER_KEYWORDS=בוט,אוטומציה,סקריפט,מתכנת,פיתוח,מחפש מפתח,דרוש מפתח,צריך עזרה,מישהו יכול,לבנות מערכת,טלגרם,אינטגרציה,אוטומטי,ווטסאפ,בהתנדבות,צ'אטבוט,צאטבוט
```

### 2.8 אישור מכשיר/IP לפני deploy — **חשוב!**

פייסבוק חוסם לוגין אוטומטי מ-IP חדש עם **security checkpoint** (דף "two_step_verification" או "checkpoint" — גם אם אין 2FA בחשבון).

**למה זה קורה:** השרת החדש = IP לא מוכר + דפדפן headless = פייסבוק דורש אימות נוסף.

**הפתרון — אישור ידני חד-פעמי:**

1. הפעל את הבוט בשרת → הוא ינסה להתחבר, ייכשל עם שגיאת checkpoint, וישמור צילום מסך `data/debug_checkpoint.png`
2. היכנס ידנית לחשבון הפייסבוק מדפדפן רגיל
3. פייסבוק ישלח התראה "ניסיון התחברות ממכשיר חדש" → **אשר**
4. הפעל שוב את הבוט → אמור להתחבר בהצלחה

**אלטרנטיבה — ייבוא session ידני:**
היכנס ידנית מדפדפן השרת, שמור cookies ל-`data/fb_session.json`.

**הערה טכנית:** הלוגין מתבצע ב-context דסקטופ (www.facebook.com) כי יש בו form HTML אמיתי. ה-cookies מועברים ל-context מובייל (m.facebook.com) לסריקה. Cookies של `.facebook.com` עובדים cross-subdomain.
